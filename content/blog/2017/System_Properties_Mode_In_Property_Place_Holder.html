--- title: "systemPropertiesModeName in Spring PropertyPlaceHolder" created: !!timestamp '2017-08-01 23:15:00' image: /media/images/blog/2015/lazare-clocks.jpg tags: - Java - Spring - Tips --- {% block excerpt %} {% mark excerpt %} PropertyPlaceholderConfigurer can help us to avoid hard code any environment specific configuration in spring configuration. {% endmark %} {% endblock %} #PropertyPlaceholderConfigurer add below bean definition in your spring xml configuration {% syntax java %} database.properties {% endsyntax %} and you can use it as below {% syntax java %} {% endsyntax %} But in this way you can only get the value from properties. what if the value is dynamic assigned. you can add a property of PropertyPlaceholderConfigurer like below {% syntax java %} {% endsyntax %} the property systemPropertiesModeName determines the order in which properties are resolved, properties file versus system properties , it can have below values SYSTEM_PROPERTIES_MODE_FALLBACK (default) Check system properties if not resolvable in the specified properties. SYSTEM_PROPERTIES_MODE_NEVER Never check system properties. SYSTEM_PROPERTIES_MODE_OVERRIDE Check system properties first, before trying the specified properties.so you can set you property dynamicly in your code using System.setProperty("key",value)
