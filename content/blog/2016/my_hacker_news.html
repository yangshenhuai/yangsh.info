I can configure which site I am going to scrape and what's my interest keywords,and the scrape service can use these data to capture the news that I am intrested, and when I want to see the news,I can just sent keyword `news` to my subscription account and it will give me the latest news of that day(my subscription account have no permission to push messages.) and all the data are persist in [redis](http://redis.io/),and you may wonder why I put all data in redis,well because I find a cool site [redis4you](http://redis4you.com/) can provide free onlie redis server(there's a [redsmin](https://redsmin.com/) provide online management for redis server),and also redis itself can provide on-disk persistence.I don't want to install any db or memery db in my computer or my VPS,because both of them hv very limited memory(1.5G and 250M). ##The configuration service I use [spring-boot](http://projects.spring.io/spring-boot/) to create configureation service. the configuration service is quite easy.it just read/update data from redis.and its very easy to integrate redis with spring-boot , you just add below to your dependencies(I use gradle) {% syntax java %} compile("org.springframework.boot:spring-boot-starter-redis:1.3.2.RELEASE") {% endsyntax %} and then you add below configurations to you `application.properties` {% syntax java %} spring.redis.host=50.30.35.9 spring.redis.password=yourpassword spring.redis.username=yourusername spring.redis.port=3464 {% endsyntax %} and its done,now you can inject the spring `RedisTemplate` to your own bean like below {% syntax java %} @Autowired public SettingService(RedisTemplate redisTemplate){ this.redisTemplate = redisTemplate; } {% endsyntax %} and for configuration, I just add a list(keyword:urls) to store the urls and for each url I will create a set for it to store the keywords I would scrape later. btw redis provide `Pipelining` to send the multiple commands to the server without waiting for the replies at all, and finally read the replies in a single step , spring `RedisTemplate` also support `Pipelining` {% syntax java %} /** * Executes the given action object within a connection that can be exposed or not.
